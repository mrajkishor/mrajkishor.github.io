

### ‚úÖ What is Context in React?

The **React Context API** provides a way to **pass data** through the component tree **without having to pass props manually at every level**.

It is mainly used for **global state** like:

* Current authenticated user
* Theme (light/dark)
* Language/localization settings

---

### üß± Steps to Create and Use Context

#### 1Ô∏è‚É£ Create the Context

```js
import React from 'react';

const ThemeContext = React.createContext();
```

#### 2Ô∏è‚É£ Create a Provider Component

This wraps part of your component tree and supplies the value.

```jsx
const ThemeProvider = ({ children }) => {
  const theme = 'dark';
  
  return (
    <ThemeContext.Provider value={theme}>
      {children}
    </ThemeContext.Provider>
  );
};
```

#### 3Ô∏è‚É£ Use Context in Consumer Components

**Method A: `useContext()` Hook (Functional Components)**

```js
import React, { useContext } from 'react';

const ThemedButton = () => {
  const theme = useContext(ThemeContext);
  return <button className={theme}>Click me</button>;
};
```

**Method B: `Context.Consumer` (Class or Functional Components)**

```jsx
<ThemeContext.Consumer>
  {theme => <button className={theme}>Click me</button>}
</ThemeContext.Consumer>
```

### ‚öôÔ∏è Full Working Example

```jsx
// ThemeContext.js
import { createContext } from 'react';
export const ThemeContext = createContext();

// ThemeProvider.js
import React, { useState } from 'react';
import { ThemeContext } from './ThemeContext';

export const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};
```

```jsx
// App.js
import React from 'react';
import { ThemeProvider } from './ThemeProvider';
import ThemedComponent from './ThemedComponent';

function App() {
  return (
    <ThemeProvider>
      <ThemedComponent />
    </ThemeProvider>
  );
}
```

```jsx
// ThemedComponent.js
import React, { useContext } from 'react';
import { ThemeContext } from './ThemeContext';

const ThemedComponent = () => {
  const { theme, setTheme } = useContext(ThemeContext);
  return (
    <div>
      <p>Current Theme: {theme}</p>
      <button onClick={() => setTheme(prev => prev === 'light' ? 'dark' : 'light')}>
        Toggle Theme
      </button>
    </div>
  );
};

export default ThemedComponent;
```

### ‚ö†Ô∏è Best Practices

| Tip                                         | Description                                                    |
| ------------------------------------------- | -------------------------------------------------------------- |
| ‚úÖ Split context creation and provider logic | Improves reusability and testability                           |
| ‚úÖ Avoid overusing context                   | For complex state, prefer Redux/Zustand                        |
| ‚úÖ Memoize provider value                    | Prevents unnecessary re-renders                                |
| ‚ö†Ô∏è Don't use for high-frequency updates     | E.g., mouse position, as it causes re-renders of all consumers |
