

## ЁЯУЭ Infix to Postfix Conversion (Reverse Polish Notation)

---

### ЁЯУМ What is Infix?

An **infix expression** is the standard mathematical notation where **operators are written between operands**.

> **Example:** `A + B * C`

But it requires **parentheses and operator precedence rules** to determine evaluation order.

---

### ЁЯУМ What is Postfix?

In **postfix notation**, also called **Reverse Polish Notation (RPN)**, **operators follow operands**.

> **Example:** `A B C * +`

Postfix **removes the need for parentheses** and respects precedence automatically.

---

### ЁЯОп Why Convert Infix to Postfix?

* Easy to evaluate using a stack
* No ambiguity due to precedence or parentheses
* Widely used in compilers and expression evaluators

---

## ЁЯЫая╕П Infix to Postfix Conversion Algorithm

We use:

* A **stack** to hold operators
* A **result string** (or array) to build the postfix expression

---

### тЬЕ Operator Precedence (Highest to Lowest)

| Precedence | Operators     | Associativity |
| ---------- | ------------- | ------------- |
| 1          | `()`          | Left to Right |
| 2          | `*`, `/`, `%` | Left to Right |
| 3          | `+`, `-`      | Left to Right |

---

### ЁЯФД Algorithm Steps

1. **Initialize** an empty stack for operators and an empty list for result.
2. For each token in the expression:

   * If **operand (A-Z, 0-9)** тЖТ append to result
   * If **left parenthesis `(`** тЖТ push to stack
   * If **right parenthesis `)`** тЖТ pop from stack to result **until `(` is found**
   * If **operator**:

     * While stack is not empty and **top of stack has higher or equal precedence**, pop to result
     * Push current operator
3. After the expression, pop all remaining operators to result.

---

### ЁЯзк Example

#### ЁЯФ╣ Infix:

```text
A + B * C - D
```

#### ЁЯФБ Postfix Conversion:

| Step | Token | Stack | Result         |
| ---- | ----- | ----- | -------------- |
| 1    | A     |       | A              |
| 2    | +     | +     | A              |
| 3    | B     | +     | A B            |
| 4    | \*    | + \*  | A B            |
| 5    | C     | + \*  | A B C          |
| 6    | -     | -     | A B C \* +     |
| 7    | D     | -     | A B C \* + D   |
| end  |       |       | A B C \* + D - |

тЬЕ **Postfix:** `A B C * + D -`

---

### ЁЯУж Example with Parentheses

#### Infix:

```text
(A + B) * (C - D)
```

#### Postfix:

```text
A B + C D - *
```

---

### тЬЕ Advantages of Postfix

* No need for parentheses
* Evaluation is faster using a stack
* Well-suited for machines and interpreters

---

### тЪая╕П Common Mistakes

* Forgetting to pop all operators at end
* Mishandling associativity for operators like `^` (right to left)
* Not handling mismatched parentheses

---

## тЬЕ Evaluation of Postfix (Brief)

To evaluate postfix:

1. Use a **stack**
2. For each token:

   * If operand тЖТ push
   * If operator тЖТ pop 2 operands, apply, push result

> Example: `3 4 2 * 1 + +` тЖТ **Result: 12**

---

## ЁЯза Real-Life Applications

* Expression evaluation in **compilers**
* Used in calculators like **HP RPN calculators**
* Foundation for building interpreters or parsers

---

## ЁЯз╛ Conclusion

Infix to postfix conversion simplifies expression evaluation by removing ambiguity related to precedence and associativity. ItтАЩs efficient, reliable, and widely used in computer science systems like interpreters, compilers, and calculators.

## тЬЕ Code Summary (Java-style)

```java
if (char is operand) тЖТ output
if (char is '(') тЖТ push to stack
if (char is ')') тЖТ pop to output until '('

    while (!stack.empty() && precedence(top) >= current):

After loop тЖТ pop remaining operators to output
```

## ЁЯУМ Core Idea

A program uses:

