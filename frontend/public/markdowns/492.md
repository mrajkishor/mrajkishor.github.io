## 🧠 What is Binary Search?

Binary Search is an **O(log n)** divide-and-conquer algorithm used to search **sorted arrays**.  
But it’s not just about finding a number — the **pattern** can be adapted to find:

> Find the first and last position of a target in a sorted array

## 🏗️ Core Structure

```js
let left = 0, right = arr.length - 1;
while (left <= right) {
  let mid = Math.floor((left + right) / 2);
  if (arr[mid] === target) return mid;
  else if (arr[mid] < target) left = mid + 1;
  else right = mid - 1;
}
return -1;
```

## 🧪 Key Binary Search Patterns

### 1️⃣ **Find Element in Sorted Array**

```js
function binarySearch(nums, target) {
  let l = 0, r = nums.length - 1;
  while (l <= r) {
    const m = Math.floor((l + r) / 2);
    if (nums[m] === target) return m;
    if (nums[m] < target) l = m + 1;
    else r = m - 1;
  }
  return -1;
}
```

🧠 Used in: LeetCode 704, most simple sorted search tasks.

---

### 2️⃣ **Lower/Upper Bound**

🟢 _Find leftmost/rightmost position of target:_

```js
// Lower Bound (First ≥ target)
while (l < r) {
  let mid = Math.floor((l + r) / 2);
  if (arr[mid] < target) l = mid + 1;
  else r = mid;
}
```

- 34. Find First and Last Position  
- 852. Peak Index in a Mountain Array

### 3️⃣ **Binary Search on Answer Space**

> You don't binary search the array — you binary search the range of _answers_.

Example: _Find minimum number of hours so Koko can eat all bananas._

```js
function minEatingSpeed(piles, h) {
  let l = 1, r = Math.max(...piles);
  while (l < r) {
    const mid = Math.floor((l + r) / 2);
    let hours = 0;
    for (let pile of piles) hours += Math.ceil(pile / mid);
    if (hours > h) l = mid + 1;
    else r = mid;
  }
  return l;
}
```

🔍 LeetCode:  
- 875. Koko Eating Bananas  
- 410. Split Array Largest Sum  
- 1283. Find the Smallest Divisor

### 4️⃣ **Search in Rotated Sorted Array**

```js
function search(nums, target) {
  let l = 0, r = nums.length - 1;
  while (l <= r) {
    const m = Math.floor((l + r) / 2);
    if (nums[m] === target) return m;

    if (nums[l] <= nums[m]) { // left half is sorted
      if (nums[l] <= target && target < nums[m]) r = m - 1;
      else l = m + 1;
    } else { // right half is sorted
      if (nums[m] < target && target <= nums[r]) l = m + 1;
      else r = m - 1;
    }
  }
  return -1;
}
```

🔍 LeetCode:  
- 33. Search in Rotated Sorted Array  
- 153. Find Minimum in Rotated Array

## 🧠 Binary Search Interview Flow:

1. Is the array **sorted**, or can I **sort** it?
2. Is there a **range of possible answers**?
3. Can I define a `isValid(x)` function?
4. Can I search for a **boundary** instead of a value?
