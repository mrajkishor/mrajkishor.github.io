

# 📌 DSA Pattern: **Is the Array Sorted?**
_A critical first question that unlocks powerful algorithmic optimizations._

---

## 🧠 Why This Matters

In DSA interviews, the moment you see the input is **sorted**, your brain should flash:  
> _“Wait, sorted input? There’s something smarter than brute-force here.”_

**Sorted inputs** often allow for:
- Efficient searching
- Pointer-based iteration
- Greedy optimizations
- Monotonicity-based logic
- Preprocessing tricks

So asking _"Is the array sorted?"_ early can shortcut your path to an O(log n) or O(n) solution.

---

## ✅ When You See a Sorted Array, Think:

| Problem Intent                      | Recommended Technique         | Complexity   |
|------------------------------------|-------------------------------|--------------|
| Searching for element/value        | **Binary Search**             | `O(log n)`   |
| Finding pair/sum/difference        | **Two Pointers**              | `O(n)`       |
| Counting ranges or prefix matches  | **Prefix Sum** / Binary Search| `O(n)` / `O(log n)` |
| Duplicate elimination              | **In-place Two Pointers**     | `O(n)`       |
| Windowed optimizations             | **Sliding Window**            | `O(n)`       |
| Rotated sorted array               | **Modified Binary Search**    | `O(log n)`   |

---

## 🧪 Examples by Pattern

### 1. **Binary Search**
> When: You need to _find_ an element or its position.

🔹 Example:  
```js
let arr = [1, 3, 5, 7, 9];
let target = 7;
```
➡️ Use classic binary search to locate `target`.

🔍 Problem: [LeetCode 704 - Binary Search](https://leetcode.com/problems/binary-search)

---

### 2. **Two Pointers**
> When: You're asked to find pairs with sum, difference, or range.

🔹 Example:  
```js
let arr = [1, 2, 3, 4, 6];
let target = 6;
```
➡️ Use two pointers `i = 0`, `j = n-1` to find if `arr[i] + arr[j] == target`.

🔍 Problem: [LeetCode 167 - Two Sum II (Sorted)](https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/)

---

### 3. **Sliding Window**
> When: You're tracking a subarray (sum, max length, etc.)

🔹 Example:  
Find the smallest window whose sum ≥ `target`.

🔍 Problem: [LeetCode 209 - Minimum Size Subarray Sum](https://leetcode.com/problems/minimum-size-subarray-sum/)

---

### 4. **Prefix Sum**
> When: You're frequently asked for sum of ranges `[i...j]`.

🔹 Idea: Preprocess prefix sums so each range sum is `O(1)` lookup.

🔍 Problem: [LeetCode 303 - Range Sum Query - Immutable](https://leetcode.com/problems/range-sum-query-immutable/)

---

### 5. **Modified Binary Search**
> When: The array is sorted but rotated or has some condition.

🔹 Example:
```js
arr = [4,5,6,7,0,1,2];  target = 0;
```
➡️ Still searchable in `O(log n)` via modified binary search.

🔍 Problem: [LeetCode 33 - Search in Rotated Sorted Array](https://leetcode.com/problems/search-in-rotated-sorted-array/)

---

## 🧠 Interview Thinking Flow:

1. 🟢 Is the array sorted?
2. 🔍 Yes? → Can I binary search something? Can I use two pointers?
3. 🛑 No? → Can I sort it first (O(n log n)) and then apply?

Example:

> "Given a sorted array, remove duplicates in-place."  
➡️ Don’t loop with a set.  
➡️ Use **slow and fast pointer technique**.

---

## ❌ Common Mistakes

- Ignoring the sorted property and using brute force.
- Sorting already sorted arrays (O(n log n) wasted).
- Using HashMaps for pair-sum problems instead of two pointers.
- Not leveraging prefix sums for range queries.

---

## 🚀 Summary:

**“Is the array sorted?”** isn’t just a yes/no question — it’s a pattern trigger.

✅ Sorted → Unlock optimized logic like binary search, two pointers, prefix sum.  
❌ Not sorted? → Can you sort it and apply those tricks?

This pattern alone can help you crack **30+ LeetCode** and **real-world interview** questions in `O(n)` or `O(log n)` instead of `O(n^2)`.
