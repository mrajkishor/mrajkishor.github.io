> JavaScript OR operator : ||

> JavaScript And operator : &&

> JavaScript nullish operator : ?? (ES20)

    let a = 1 || 4 || 544 || 4;
    console.log(a); // 1
    let a = 1 && 2 && 4 && 454;
    console.log(a); // 454
    let a = undefined || undefined || 544 || undefined; // or operator stops when it finds a truthy value and returns it. 
    console.log(a); // 544
    let a = 44 && 334 && undefined && 443; // and operator stops when it finds a falsy value and returns it. 
    console.log(a); // undefined
    let a = undefined ?? 334 ?? undefined ?? 443;
    console.log(334); //nullish operator is for es20 and it stops when it finds a truthy value and returns it just like || 

But here is a catch, 

false, 0, null, NaN, undefined these values are considered as [Falsy](https://developer.mozilla.org/en-US/docs/Glossary/Falsy) by OR operator 
whereas
null and undefined only considered as Falsy by nullish oeprator. 

i.e.

    let a = false || 334 || undefined || 443;
    let a = 0 || 334 || undefined || 443;
    let a = null || 334 || undefined || 443;
    let a = undefined || 334 || undefined || 443;
    let a = NaN || 334 || undefined || 443;

these all returns 334. 

but,

    let a = false ?? 334 ?? undefined ?? 443;
    let a = 0 ?? 334 ?? undefined ?? 443;
    let a = null ?? 334 ?? undefined ?? 443;
    let a = undefined || 334 || undefined || 443;
    let a = NaN ?? 334 ?? undefined ?? 443;

these all returns false, 0, 334,  334 and NaN respectively.
