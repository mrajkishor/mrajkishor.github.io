

To solve data structure and algorithm problems with optimal time complexities like O(n) or O(log n), several techniques and patterns can be applied. Here are some of the common ones:

### O(n) Solutions:

1.  **Linear Traversal:** Iterate through the data structure once, processing each element. Commonly used for array manipulation problems.
**Example**: Calculating the sum of elements in an array.
```java
	public int calculateSum(int[] arr) {
		int total = 0;
		for (int num : arr) {
			total += num;
		}
		return total;
	}
```
2.  **Two Pointers:** Use two pointers to traverse an array from different ends or locations simultaneously. Often used in sorting or searching algorithms.
**Example**: Finding pairs in a sorted array whose sum is equal to a given target.
```java
	public int[] twoSum(int[] arr, int target) {
		int left = 0, right = arr.length - 1;
		while (left < right) {
			int currentSum = arr[left] + arr[right];
			if (currentSum == target) {
				return new int[]{arr[left], arr[right]};
			} else if (currentSum < target) {
				left++;
			} else {
				right--;
			}
		}
		return new int[0]; // No pair found
	}
```
3.  **Hashing:** Utilize hash tables or hash maps to store and retrieve elements efficiently. Useful for constant-time lookups and frequently used in problems requiring frequency counting or tracking elements. Space complexity will be O(n) due to use of Hash table.
**Example**: Finding duplicate elements in an array.
```java
	public List<Integer> findDuplicates(int[] arr) {
		List<Integer> duplicates = new ArrayList<>();
		Set<Integer> set = new HashSet<>();
		for (int num : arr) {
			if (!set.add(num)) {
				duplicates.add(num);
			}
		}
		return duplicates;
	}
```
4.  **Sliding Window:** Maintain a subset of contiguous elements in an array, moving the window based on certain conditions. Useful in problems involving strings or arrays.
**Example**: Finding the maximum sum of a subarray of length k
```java
	public int maxSubarraySum(int[] arr, int k) {
		int windowSum = 0;
		for (int i = 0; i < k; i++) {
			windowSum += arr[i];
		}
		int maxSum = windowSum;
		for (int i = k; i < arr.length; i++) {
			windowSum += arr[i] - arr[i - k];
			maxSum = Math.max(maxSum, windowSum);
		}
		return maxSum;
	}
```



### O(log n) Solutions:

1.  **Binary Search:** Divide the search space in half at each step, discarding one half based on the property of the problem. Applicable to sorted arrays or other data structures with sorted properties.
**Example**: Searching for an element in a sorted array.
```java
	public int binarySearch(int[] arr, int target) {
		int low = 0, high = arr.length - 1;
		while (low <= high) {
			int mid = low + (high - low) / 2;
			if (arr[mid] == target) {
				return mid;
			} else if (arr[mid] < target) {
				low = mid + 1;
			} else {
				high = mid - 1;
			}
		}
		return -1; // Element not found
	}
```
2.  **Binary Tree Operations:** Utilize binary tree properties to perform operations such as searching, inserting, or deleting elements efficiently. Techniques like balancing trees (e.g., AVL trees, Red-Black trees) maintain logarithmic height for efficient operations.
**Example**: Searching for a value in a Binary Search Tree (BST).
```java
	class TreeNode {
		int val;
		TreeNode left;
		TreeNode right;
		TreeNode(int val) {
			this.val = val;
		}
	}

	public TreeNode searchBST(TreeNode root, int val) {
		if (root == null || root.val == val) {
			return root;
		}
		if (val < root.val) {
			return searchBST(root.left, val);
		} else {
			return searchBST(root.right, val);
		}
	}
```

3.  **Divide and Conquer:** Break down a problem into smaller subproblems, solve them independently, and combine the solutions to get the final result. Algorithms like merge sort and quicksort use this approach.
**Example**: Merge Sort - sorting an array using the divide and conquer approach.
```java
	public void mergeSort(int[] arr, int left, int right) {
		if (left < right) {
			int mid = left + (right - left) / 2;
			mergeSort(arr, left, mid);
			mergeSort(arr, mid + 1, right);
			merge(arr, left, mid, right);
		}
	}

	private void merge(int[] arr, int left, int mid, int right) {
		int[] temp = new int[right - left + 1];
		int i = left, j = mid + 1, k = 0;
		while (i <= mid && j <= right) {
			if (arr[i] <= arr[j]) {
				temp[k++] = arr[i++];
			} else {
				temp[k++] = arr[j++];
			}
		}
		while (i <= mid) {
			temp[k++] = arr[i++];
		}
		while (j <= right) {
			temp[k++] = arr[j++];
		}
		System.arraycopy(temp, 0, arr, left, temp.length);
	}
```


### General Techniques for Optimization:

1.  **Optimizing Loops:** Analyze loops to reduce unnecessary iterations or nested loops. Use techniques like early termination or skipping unnecessary iterations to improve efficiency.
2.  **Space-Time Tradeoffs:** Sometimes sacrificing space for time or vice versa can lead to optimized solutions. Precomputing results, memoization, or caching can help in certain scenarios.
3.  **Dynamic Programming:** Solve problems by breaking them down into simpler subproblems and storing the results of these subproblems to avoid redundant computations. Often used in problems with overlapping substructures.
4.  **Greedy Algorithms:** Make locally optimal choices at each stage with the hope of finding a global optimum. Not always applicable, but can lead to efficient solutions for certain problems.

Additionally, understanding the properties of various data structures (arrays, linked lists, stacks, queues, trees, graphs, etc.) and their time complexities for different operations is crucial for choosing the right approach to achieve optimal solutions in terms of time complexity.
