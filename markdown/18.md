The `HashMap` and `ConcurrentHashMap` are both implementations of the `Map` interface in Java, but they are designed for different use cases, particularly with regard to thread safety and performance.

### HashMap

- **Thread-Safety**: Not thread-safe. If multiple threads access a `HashMap` concurrently and at least one of the threads modifies the map structurally, it must be synchronized externally.
- **Performance**: Generally faster than `ConcurrentHashMap` because it doesn't need to handle synchronization overhead.
- **Use Case**: Use `HashMap` when you have a single-threaded environment or when thread safety is not a concern.

#### Example Usage of HashMap

```java
import java.util.HashMap;
import java.util.Map;

public class HashMapExample {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();
        map.put("A", 1);
        map.put("B", 2);
        map.put("C", 3);

        // Iterating over the HashMap
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue());
        }
    }
}
```

### ConcurrentHashMap

- **Thread-Safety**: Thread-safe. It is designed for concurrent access from multiple threads. It achieves thread safety with a combination of locking and partitioning (locking only part of the map during updates).
- **Performance**: Generally slower than `HashMap` due to the overhead of ensuring thread safety, but it offers better performance under concurrent access compared to synchronizing a `HashMap`.
- **Use Case**: Use `ConcurrentHashMap` when you need to handle concurrent access and modifications by multiple threads without using synchronized blocks or methods.

#### Example Usage of ConcurrentHashMap

```java
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ConcurrentHashMapWordCount {
    public static void main(String[] args) {
        // Shared ConcurrentHashMap to store word counts
        ConcurrentHashMap<String, Integer> wordCountMap = new ConcurrentHashMap<>();

        // Sample documents (for simplicity, using arrays of words)
        String[] document1 = {"hello", "world", "hello"};
        String[] document2 = {"hello", "java"};
        String[] document3 = {"java", "concurrent", "world"};

        // ExecutorService to manage threads
        ExecutorService executor = Executors.newFixedThreadPool(3);

        // Runnable tasks to count words in each document
        Runnable task1 = () -> countWords(document1, wordCountMap);
        Runnable task2 = () -> countWords(document2, wordCountMap);
        Runnable task3 = () -> countWords(document3, wordCountMap);

        // Execute tasks
        executor.execute(task1);
        executor.execute(task2);
        executor.execute(task3);

        // Shutdown executor
        executor.shutdown();
        while (!executor.isTerminated()) {
            // Wait until all tasks are finished
        }

        // Print the word counts
        wordCountMap.forEach((word, count) -> System.out.println(word + ": " + count));
    }

    // Method to count words and update the ConcurrentHashMap
    private static void countWords(String[] document, ConcurrentHashMap<String, Integer> wordCountMap) {
        for (String word : document) {
            wordCountMap.merge(word, 1, Integer::sum);
        }
    }
}
```

### Key Differences and When to Use

- **Single-Threaded Environment**: Use `HashMap` for better performance.
- **Multi-Threaded Environment**: Use `ConcurrentHashMap` to avoid concurrency issues and ensure thread-safe operations.
- **Synchronized HashMap Alternative**: If you need a thread-safe map but don't want to use `ConcurrentHashMap`, you can synchronize a `HashMap` using `Collections.synchronizedMap(new HashMap<>())`, but `ConcurrentHashMap` typically provides better performance for concurrent access scenarios.
