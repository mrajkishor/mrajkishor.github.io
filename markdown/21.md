

Managing asynchronous operations in a Redux application can be challenging. Thankfully, tools like Redux Toolkit and Redux Saga provide robust solutions. In this blog, we will explore how to make API calls using both Redux Toolkit and Redux Saga, compare their approaches, and determine which one might be better suited for your application.

### API Calls with Redux Toolkit

Redux Toolkit (RTK) simplifies Redux development with utilities for reducing boilerplate code. It includes `createSlice` for managing state and `createAsyncThunk` for handling async logic.

#### Setting Up Redux Toolkit

1. **Install Redux Toolkit:**
   ```bash
   npm install @reduxjs/toolkit react-redux
   ```

2. **Create an Async Thunk:**
   ```javascript
   // features/user/userSlice.js
   import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';

   export const fetchUser = createAsyncThunk(
     'user/fetchUser',
     async (userId) => {
       const response = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`);
       return response.json();
     }
   );

   const userSlice = createSlice({
     name: 'user',
     initialState: {
       data: null,
       status: 'idle',
       error: null,
     },
     reducers: {},
     extraReducers: (builder) => {
       builder
         .addCase(fetchUser.pending, (state) => {
           state.status = 'loading';
         })
         .addCase(fetchUser.fulfilled, (state, action) => {
           state.status = 'succeeded';
           state.data = action.payload;
         })
         .addCase(fetchUser.rejected, (state, action) => {
           state.status = 'failed';
           state.error = action.error.message;
         });
     },
   });

   export default userSlice.reducer;
   ```

3. **Configure the Store:**
   ```javascript
   // app/store.js
   import { configureStore } from '@reduxjs/toolkit';
   import userReducer from '../features/user/userSlice';

   export const store = configureStore({
     reducer: {
       user: userReducer,
     },
   });
   ```

4. **Use the Thunk in a Component:**
   ```javascript
   // components/UserComponent.js
   import React, { useEffect } from 'react';
   import { useDispatch, useSelector } from 'react-redux';
   import { fetchUser } from '../features/user/userSlice';

   const UserComponent = ({ userId }) => {
     const dispatch = useDispatch();
     const user = useSelector((state) => state.user.data);
     const status = useSelector((state) => state.user.status);
     const error = useSelector((state) => state.user.error);

     useEffect(() => {
       dispatch(fetchUser(userId));
     }, [dispatch, userId]);

     if (status === 'loading') return <div>Loading...</div>;
     if (status === 'failed') return <div>Error: {error}</div>;

     return user ? (
       <div>
         <h1>{user.name}</h1>
         <p>{user.email}</p>
       </div>
     ) : (
       <div>No user data</div>
     );
   };

   export default UserComponent;
   ```

### API Calls with Redux Saga

Redux Saga is a middleware library that handles side effects in Redux. It uses ES6 generators to manage async flows.

#### Setting Up Redux Saga

1. **Install Redux Saga:**
   ```bash
   npm install redux-saga
   ```

2. **Create a Saga:**
   ```javascript
   // sagas/userSaga.js
   import { call, put, takeLatest } from 'redux-saga/effects';

   function* fetchUser(action) {
     try {
       const response = yield call(fetch, `https://jsonplaceholder.typicode.com/users/${action.payload}`);
       const data = yield response.json();
       yield put({ type: 'user/fetchUserSuccess', payload: data });
     } catch (error) {
       yield put({ type: 'user/fetchUserFailure', payload: error.message });
     }
   }

   function* userSaga() {
     yield takeLatest('user/fetchUserRequest', fetchUser);
   }

   export default userSaga;
   ```

3. **Configure the Store with Saga Middleware:**
   ```javascript
   // app/store.js
   import { createStore, applyMiddleware } from 'redux';
   import createSagaMiddleware from 'redux-saga';
   import userReducer from '../reducers/userReducer';
   import userSaga from '../sagas/userSaga';

   const sagaMiddleware = createSagaMiddleware();

   const store = createStore(
     userReducer,
     applyMiddleware(sagaMiddleware)
   );

   sagaMiddleware.run(userSaga);

   export default store;
   ```

4. **Create a Reducer:**
   ```javascript
   // reducers/userReducer.js
   const initialState = {
     data: null,
     status: 'idle',
     error: null,
   };

   const userReducer = (state = initialState, action) => {
     switch (action.type) {
       case 'user/fetchUserRequest':
         return { ...state, status: 'loading' };
       case 'user/fetchUserSuccess':
         return { ...state, status: 'succeeded', data: action.payload };
       case 'user/fetchUserFailure':
         return { ...state, status: 'failed', error: action.payload };
       default:
         return state;
     }
   };

   export default userReducer;
   ```

5. **Dispatch Actions in a Component:**
   ```javascript
   // components/UserComponent.js
   import React, { useEffect } from 'react';
   import { useDispatch, useSelector } from 'react-redux';

   const UserComponent = ({ userId }) => {
     const dispatch = useDispatch();
     const user = useSelector((state) => state.data);
     const status = useSelector((state) => state.status);
     const error = useSelector((state) => state.error);

     useEffect(() => {
       dispatch({ type: 'user/fetchUserRequest', payload: userId });
     }, [dispatch, userId]);

     if (status === 'loading') return <div>Loading...</div>;
     if (status === 'failed') return <div>Error: {error}</div>;

     return user ? (
       <div>
         <h1>{user.name}</h1>
         <p>{user.email}</p>
       </div>
     ) : (
       <div>No user data</div>
     );
   };

   export default UserComponent;
   ```

### Comparison: Redux Toolkit vs Redux Saga

#### Redux Toolkit:
- **Pros:**
  - Simplifies Redux setup and reduces boilerplate.
  - Integrates well with modern React patterns.
  - Provides `createAsyncThunk` for handling async actions easily.
  - Good for straightforward async logic and side effects.
- **Cons:**
  - Less control over complex async flows and side effects compared to sagas.

#### Redux Saga:
- **Pros:**
  - Provides fine-grained control over async logic and side effects.
  - Uses generator functions, making it powerful for complex workflows.
  - Easy to test due to the use of generator functions.
- **Cons:**
  - More boilerplate and setup compared to Redux Toolkit.
  - Steeper learning curve, especially for developers unfamiliar with generators.

### Which One is Better?

The choice between Redux Toolkit and Redux Saga depends on your application's requirements:

- **Use Redux Toolkit if:**
  - You want a simple, streamlined way to handle async logic.
  - Your async operations are not overly complex.
  - You prefer less boilerplate and a more modern approach.

- **Use Redux Saga if:**
  - You need fine-grained control over complex async flows.
  - Your application has complex side effects that need to be managed meticulously.
  - You value the testability of async logic through generator functions.

For most straightforward use cases, Redux Toolkit is the better choice due to its simplicity and ease of use. However, for applications with more complex requirements, Redux Saga provides the necessary power and flexibility to handle intricate async workflows effectively.

> Winner : Redux Saga
